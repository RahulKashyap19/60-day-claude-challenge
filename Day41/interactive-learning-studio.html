Foundations — Learning From Labeled Examples

foundational

What supervised learning is, and the fork in the road: number or category?

Supervised learning is the branch of machine learning where you teach a model using examples that already have the "correct answer" attached. Each training example is a pair: a set of features (inputs describing the thing) and a label (the answer you want the model to eventually predict on its own). The model's job is to find a pattern that maps features to labels well enough to generalize to new, unseen examples.

Think of it like a new analyst joining your delivery team with a binder of 500 past project sign-offs, each one marked "on time" or "delayed," along with the project's team size, scope changes, and client region. Given enough solved examples, the analyst starts noticing patterns — before you ever ask them to predict anything about a brand-new project.

Every supervised learning problem falls into one of two families, depending on what kind of answer you're predicting:

Regression

The label is a continuous number. "How much?" or "How many?" — house price, delivery time in days, monthly revenue, churn probability score as a raw number.

Classification

The label is a category from a fixed set. "Which one?" — spam or not spam, will-churn or won't-churn, malignant or benign, or one of several ticket priority levels.

Diagram — the shape of the two problems

Left: regression fits a line (or curve) through the cloud of points to predict a value anywhere along it. Right: classification finds a boundary that separates groups so a new point can be assigned to a side.

Key vocabulary

TermMeaning
FeatureAn input variable used to make the prediction (e.g. square footage, number of tickets raised)
Label / TargetThe known correct answer in the training data (price, churned yes/no)
Training setLabeled examples the model learns from
Test setHeld-back labeled examples used only to check how well the model generalizes
ModelThe learned function that maps features → predicted label
Prediction / InferenceRunning the trained model on a new, unlabeled example
Misconception: "Classification always means picking between exactly two options." Not true — this is called binary classification, but multi-class classification (e.g. routing a support ticket to one of six teams) and multi-label classification (an example can belong to more than one category at once) are both common and both still "classification."
Misconception: "Regression means fitting a straight line." Linear regression fits a straight line, but regression as a category includes curved and highly non-linear fits too — the defining trait is a continuous numeric output, not the shape of the fit.

Key takeaways
  • Supervised learning always starts with labeled examples: features in, known answer attached.
  • Regression predicts a continuous number; classification predicts a category from a fixed set.
  • The type of label you're predicting — not the algorithm's complexity — determines which family you're in.
  • Training data teaches the model; test data tells you if it actually generalizes.
