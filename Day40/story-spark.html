The system prompt

Story Spark's entire job is to invent its own constraints — genre, setting, character, twist — and then write into them. The prompt tells the model to silently pick three "seed ingredients" itself and treat them as its own private writing prompt, then surface them afterward as a fun footer rather than showing its work up front.

It replies in strict JSON so the frontend can separate the title, genre tag, seed ingredients, and prose into distinct styled zones instead of parsing one blob of freeform text.

Handling edge cases

  • No user input is required, so there's nothing to validate on the way in — but the app still sends a short list of recently-used genres/titles as context, and the prompt is instructed to actively avoid repeating them, so "Another spark" doesn't feel like the same three stories on rotation.
  • If that recent-history text ever contained something unrelated or manipulative, the prompt treats it strictly as "titles to avoid," never as new instructions — the same data-not-commands rule any user-facing field should get.
  • The model is explicitly kept PG-13 and all-ages friendly: no graphic violence, no explicit content, even if a random dark seed (say, "a funeral") comes up — the whimsical tone is a hard constraint, not just a suggestion.
  • If the API call fails or returns something that isn't valid JSON, the UI shows a dedicated error state with a retry button rather than a broken or blank page.

Why this UI

Casual writers with "low commitment" don't want a form — they want a slot machine. So there's exactly one button on first load. The result renders as an actual storybook page (drop cap included) sitting on a colorful studio background, because the payoff should look like a finished little artifact, not a chat reply.

The seed-ingredient tags at the bottom of the page turn the randomness into a visible, shareable detail instead of hiding it — it's the same instinct as showing your dice roll in a board game.

How to extend it

  • Tools: add an image-generation tool so each spark gets a small generated cover illustration alongside the text.
  • Memory: persist the saved shelf server-side per user instead of per session, so favorites survive a reload.
  • Multi-step flow: add a "keep writing" action that sends the generated opening back to Claude with an instruction to continue it another 300 words, turning a spark into a short story session.
  • Light input: reintroduce an optional single genre chip row for people who do want to nudge the randomness, without losing the one-click default.

Every spark is generated fresh — nothing here is a real place, person, or event.
